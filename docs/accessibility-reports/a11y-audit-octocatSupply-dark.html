The primary color #76B852 used for text and accents on the dark background #0A0A0A produces a contrast ratio of approximately 5.5:1. export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');

  // Add keyboard shortcut listener
  useEffect(() => {
    const handleKeyboard = (e: KeyboardEvent) => {
      // Ctrl/Cmd + Shift + D to toggle theme
      if ((e.ctrlKey || e.metaKey) && e.shiftKey && e.key === 'D') {
        e.preventDefault();
        toggleTheme();
      }
    };

    window.addEventListener('keydown', handleKeyboard);
    return () => window.removeEventListener('keydown', handleKeyboard);
  }, []);

}

1.4.12 Text Spacing - Custom Stylesheet Compatibility

WCAG 2.2 Level AA ‚Üó

üìç Location

All components with dark mode styling

üîç Issue Description

WCAG 2.2 requires that content remains readable when users apply custom text spacing via browser extensions or stylesheets. The following spacing adjustments should not cause text truncation or overlap:

  • Line height to at least 1.5 times the font size
  • Paragraph spacing to at least 2 times the font size
  • Letter spacing to at least 0.12 times the font size
  • Word spacing to at least 0.16 times the font size

‚ö° Impact

Users with dyslexia or reading disabilities who apply custom spacing settings may experience text overflow or clipping in dark mode components.

üí° Recommended Enhancement

Testing Procedure:

  1. Install a text spacing bookmarklet or browser extension
  2. Apply maximum spacing adjustments in dark mode
  3. Check all components for text clipping or overlap
  4. Verify fixed-height containers adapt to increased spacing

Preventative CSS Pattern:

frontend/src/index.css 
/* Ensure components adapt to custom spacing */
.dark {
  /* Use min-height instead of fixed height */
  min-height: auto;
  
  /* Avoid fixed line-height on constrained containers */
  line-height: inherit; /* Don't override user styles */
}

/* Test with this spacing override */
* {
  line-height: 1.5 !important;
  letter-spacing: 0.12em !important;
  word-spacing: 0.16em !important;
}
p {
  margin-bottom: 2em !important;
}

Components to Verify:

  • Navigation menu items
  • Cart item descriptions
  • Product cards
  • Form labels and inputs
  • Footer text

Dark Mode Testing Recommendations

  • Automated Contrast Testing:

    Run automated tools like axe DevTools, WAVE, or Lighthouse in dark mode to catch contrast violations. Test actual rendered colors, not just CSS values, as anti-aliasing and font rendering affect perceived contrast.

  • High Contrast Mode Validation:

    Test with Windows High Contrast themes and forced-colors media query. Ensure custom colors don't override system color preferences and all UI components remain visible.

  • Keyboard Navigation Audit:

    Tab through the entire application in dark mode without using a mouse. Verify all focus indicators are visible and have sufficient contrast. Test with macOS VoiceOver or NVDA to confirm focus tracking.

  • Multi-Device Testing:

    View the application on different displays (laptop, external monitor, mobile devices) to verify contrast holds up across varying screen qualities and brightness levels. Test with actual #0A0A0A background in different ambient lighting.

  • Theme Persistence Verification:

    Verify theme preference persists correctly with localStorage, respects prefers-color-scheme media query, and provides a smooth non-blocking theme transition. Ensure no FOUC (flash of unstyled content) occurs.

  • Color Blindness Simulation:

    Use tools like Color Oracle or browser DevTools to simulate color blindness (protanopia, deuteranopia, tritanopia). Verify the green primary color remains distinguishable from gray backgrounds across all color vision types.
