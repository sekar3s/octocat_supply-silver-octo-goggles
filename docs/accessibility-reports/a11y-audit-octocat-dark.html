Critical focus visibility and navigation issues prevent WCAG 2.2 Level AA compliance in dark mode.

Dark Mode Context

Theme Configuration

  • Background: #0A0A0A (Near black)
  • Text: #F5F5F5 (Off-white) - 15:1 contrast ratio ‚úì
  • Primary: #76B852 (Green) - More visible on dark backgrounds
  • Implementation: React Context with CSS custom properties

Dark Mode Specific Challenges: While text contrast is excellent, focus indicators, boundaries, and interactive states face visibility challenges unique to dark backgrounds. Default browser focus rings (typically dark blue/black) become invisible against #0A0A0A.

Level A - Critical Issues

These issues violate WCAG 2.2 Level A success criteria and prevent basic accessibility. They are even more severe in dark mode due to contrast visibility.

No Screen Reader Announcements for Route Changes

üìã WCAG 4.1.3 Status Messages (AA) ‚Üí üîó

‚úÖ Add Route Announcer
RouteAnnouncer.tsx 
import { useEffect } from 'react';
import { useLocation } from 'react-router-dom';

function RouteAnnouncer() {
  const location = useLocation();

  useEffect(() => {
    // Get page title or route name
    const pageTitle = document.title || 'Page';
    
    // Announce to screen readers
    const announcement = `Navigated to ${pageTitle}`;
    const announcer = document.getElementById('route-announcer');
    
    if (announcer) {
      announcer.textContent = announcement;
    }
  }, [location]);

  return (
    <div
      id="route-announcer"
      role="status"
      aria-live="polite"
      aria-atomic="true"
      className="sr-only"
    />
  );
}

// Add to App.tsx
function App() {
  return (
    <>
      <RouteAnnouncer />
      <Outlet />
    </>
  );
}

‚úÖ Recommended Approach

Safe HTML Rendering 
import DOMPurify from 'dompurify';

// Sanitize and enforce semantic structure
function SafeHTML({ html }) {
  const clean = DOMPurify.sanitize(html, {
    ALLOWED_TAGS: ['p', 'h2', 'h3', 'ul', 'li', 'strong', 'em', 'a'],
    ALLOWED_ATTR: ['href', 'class', 'id', 'aria-label']
  });

  return (
    <div 
      dangerouslySetInnerHTML={{ __html: clean }}
      className="prose prose-invert" /* Tailwind typography */
    />
  );
}
